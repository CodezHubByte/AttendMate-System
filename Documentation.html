AttendMate - Project Documentation

A comprehensive attendance management system built with PHP, HTML, CSS, and JavaScript.

1. About AttendMate

AttendMate is designed to streamline the process of managing student attendance for educational institutions. It provides an intuitive interface for administrators to mark and view attendance, manage student records, and generate reports. Students can also view their attendance records and stay informed through announcements.

This project aims to provide a reliable and efficient solution for tracking attendance, reducing manual errors, and improving communication between faculty and students.

2. Features

AttendMate offers a robust set of features categorized by user roles:

Admin Features:

  • Dashboard: A centralized overview of system activities and quick access to key functionalities.
  • Manage Students: CRUD operations (Create, Read, Update, Delete) for student records, including adding new students, editing profiles, and removing inactive accounts.
  • Mark Attendance: Efficient system for recording daily attendance for various classes/batches.
  • View Attendance: Detailed view of attendance records, filterable by date, student, or class.
  • Attendance Reports: Generate insightful reports (e.g., daily, weekly, monthly summaries, absentee lists) for analysis.
  • Announcements: Broadcast important messages and updates to all students.
  • Remark System: Ability to add remarks or notes related to student performance or behavior.
  • View All Remarks: Centralized view of all remarks made by administrators.

Student Features:

  • Dashboard: A personalized overview showing quick stats and important notifications.
  • My Attendance: Students can view their own attendance history and overall attendance percentage.
  • My Profile: View and update personal profile information.
  • Announcements: Receive and read announcements posted by administrators.
  • Change Password: Securely update login credentials.

Additionally, the system includes Role-Based Access Control to ensure that users only access functionalities relevant to their permissions (Admin vs. Student).

3. Technologies Used

AttendMate is built using a combination of modern web technologies:

  • Frontend:
    • HTML5: For structuring the web content.
    • CSS3: For styling and responsive design, enhanced with custom CSS.
    • JavaScript: For interactive elements and dynamic content.
    • Bootstrap 5: A powerful front-end framework for responsive and mobile-first design.
    • Font Awesome 6: For scalable vector icons.
  • Backend:
    • PHP: Server-side scripting language for logic, database interaction, and session management.
  • Database:
    • MySQL: A relational database management system for storing application data.

4. Setup and Installation

To get AttendMate up and running on your local machine, follow these steps:

4.1. Prerequisites

Ensure you have the following software installed on your system:

  • Web Server with PHP: A local server environment like XAMPP, WAMP, or MAMP is recommended, as it includes Apache, PHP, and MySQL.
  • MySQL Database: Comes bundled with XAMPP/WAMP/MAMP.
  • Git: For cloning the repository (Download Git).

4.2. Database Setup

  1. Start your Apache and MySQL services from your XAMPP/WAMP/MAMP control panel.
  2. Open your web browser and go to `http://localhost/phpmyadmin/`.
  3. Create a new database named `attendmate_db`.
  4. Import the provided SQL file to set up the necessary tables and initial data.

    5. Note: You will find the `database.sql` file in the root of the project repository. Select the `attendmate_db` database, go to the "Import" tab, choose the `database.sql` file, and click "Go". 

    -- Example SQL content (you'll put your actual schema here)
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    role ENUM('admin', 'user') NOT NULL
);

CREATE TABLE students (
    student_id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT UNIQUE,
    enrollment_number VARCHAR(100) UNIQUE NOT NULL,
    -- ... other student details
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

-- And so on for attendance, announcements, remarks tables.
-- Include an INSERT statement for a default admin user.
INSERT INTO users (name, email, password, role) VALUES ('Admin User', 'admin@attendmate.com', 'adminpassword', 'admin');

4.3. Project Files Setup

  1. Clone the repository: Open your terminal or Git Bash and run: 
    git clone https://github.com/CodezHubByte/AttendMate-System
  2. Place project files:
    • If using XAMPP, move the cloned `AttendMate` folder into the `htdocs` directory (e.g., `C:\xampp\htdocs\AttendMate`).
    • If using WAMP, move it into the `www` directory (e.g., `C:\wamp\www\AttendMate`).
  3. Configure Database Connection:

    Locate the database connection file (e.g., `includes/db_connect.php` or `config.php`) and update the database credentials if necessary. Example: 

    <?php
$servername = "localhost";
$username = "root"; // Your MySQL username
$password = ""; // Your MySQL password (often empty for XAMPP/WAMP)
$dbname = "attendmate_db"; // The database name you created

// Create connection
$conn = new mysqli($servername, $username, $password, $dbname);

// Check connection
if ($conn->connect_error) {
    die("Connection failed: " . $conn->connect_error);
}
?>

4.4. Accessing the Application

Once all steps are completed, open your web browser and navigate to: 

http://localhost/AttendMate/index.php

You should see the login page for AttendMate.

5. Usage Guide

Here’s how to use AttendMate after successful installation:

5.1. Login Credentials

  • Admin Default Credentials:
    • Email: admin@attendmate.com
    • Password: The password you set for the admin user in `database.sql` file
  • Student Credentials:

    Students will log in using their registered email/username and password, which can be managed by the admin or created during student registration.

5.2. Admin Workflow

  1. Dashboard: After logging in, you'll see the admin dashboard with an overview.
  2. Manage Students: Use this section to add new students, edit their details, or remove them from the system.
  3. Mark Attendance: Navigate to this page to record attendance for specific classes/dates.
  4. View Attendance: Review attendance records for individual students or entire classes.
  5. Attendance Reports: Generate various reports to track attendance patterns and statistics.
  6. Announcements: Post new announcements that will be visible to all students on their dashboards.
  7. Remark: Add specific remarks for students.
  8. View Remarks: See all recorded remarks.

5.3. Student Workflow

  1. Dashboard: Upon logging in, students see their personalized dashboard.
  2. My Attendance: View their attendance history and current attendance percentage.
  3. My Profile: Access and update personal information.
  4. Announcements: Read announcements posted by the administration.
  5. Change Password: Update their account password.
